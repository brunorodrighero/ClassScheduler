# Contributing to Class Scheduler / Agendador de Aulas

Firstly, thank you for your interest in contributing to our project! Your support and assistance are greatly appreciated. In order to ensure that contributions go smoothly, we've established some guidelines.

## Guidelines

### Issues

If you've discovered a bug or have a feature that you'd like to suggest, please [create an issue](https://github.com/brunorodrighero/ClassScheduler/issues/new) in the GitHub repository. Be as clear and descriptive as possible.

### Pull Requests

If you'd like to contribute code to fix a bug, implement a feature, or enhance the project in some way, please fork the repository and submit a pull request.

Before you submit, please ensure that:

- The code compiles and all tests pass.
- The code adheres to the existing coding conventions within the project.

# Contribuindo para o Agendador de Aulas / Class Scheduler

Primeiramente, obrigado pelo seu interesse em contribuir para o nosso projeto! Seu apoio e assistência são muito apreciados. Para garantir que as contribuições corram bem, estabelecemos algumas diretrizes.

## Diretrizes

### Questões

Se você descobriu um bug ou tem uma funcionalidade que gostaria de sugerir, por favor [crie uma issue](https://github.com/brunorodrighero/ClassScheduler/issues/new) no repositório GitHub. Seja o mais claro e descritivo possível.

### Pull Requests

Se você gostaria de contribuir com código para corrigir um bug, implementar uma funcionalidade ou melhorar o projeto de alguma forma, por favor faça um fork do repositório e submeta um pull request.

Antes de enviar, por favor, certifique-se de que:

- O código compila e todos os testes passam.
- O código adere às convenções de codificação existentes dentro do projeto.
